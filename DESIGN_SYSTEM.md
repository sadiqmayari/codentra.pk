# Codentra Design System

This document defines the visual language and UI rules for Codentra.pk.
All UI components must follow these standards for consistency and premium feel.

---

## 🎯 Design Philosophy

- Minimal but premium
- SaaS-grade clarity (Stripe / Vercel inspired)
- Strong typography hierarchy
- Intentional whitespace
- Conversion-focused UI (every section has a purpose)

---

## 🎨 Color System

### Primary Palette (Brand Core)
- Primary: #4F46E5 (Indigo - trust + tech)
- Primary Dark: #3730A3
- Primary Light: #818CF8

---

### Neutral Palette
- Background: #0B0F19 (dark mode base)
- Surface: #111827
- Card: #1F2937
- Border: #2D3748

---

### Text Colors
- Primary Text: #F9FAFB
- Secondary Text: #9CA3AF
- Muted Text: #6B7280

---

### Accent Colors
- Success: #10B981
- Warning: #F59E0B
- Error: #EF4444
- Info: #3B82F6

---

## 📏 Spacing System

Use a consistent 8px grid system.

### Spacing Scale
- 4px → xs
- 8px → sm
- 16px → md
- 24px → lg
- 32px → xl
- 48px → 2xl
- 64px → 3xl
- 96px → 4xl

---

## 🧱 Layout Rules

- Max content width: 1200px
- Section padding: 80px vertical (desktop)
- Section padding: 48px vertical (mobile)
- Container padding: 24px horizontal
- Always center main content

---

## ✍️ Typography System

### Font Family
- Primary: Inter (clean SaaS standard)
- Optional fallback: system-ui, sans-serif

---

### Type Scale

- H1: 48–64px (Hero headlines only)
- H2: 32–40px (Section headers)
- H3: 24–28px (Sub-sections)
- Body: 16–18px
- Small Text: 14px
- Caption: 12px

---

### Typography Rules

- H1 must be bold and emotionally strong
- H2 must describe value, not just title
- Body text must be readable with high contrast
- Line height:
  - Headings: 1.1–1.3
  - Body: 1.6–1.8

---

## 🧩 UI Component Principles

- Buttons must have clear hierarchy:
  - Primary CTA (filled, high contrast)
  - Secondary (outline)
  - Ghost (minimal)

- Cards:
  - Soft borders or subtle shadows
  - Rounded corners (12–16px)
  - No heavy shadows

- Inputs:
  - Clean borders
  - Focus state clearly visible
  - No clutter

---

## 🌫️ Effects & Style Rules

- Use subtle gradients only (never heavy neon)
- Hover states must be smooth (200–300ms transitions)
- Avoid excessive animations
- Micro-interactions only for UX clarity

---

## 📱 Responsiveness Rules

- Mobile-first design
- Breakpoints:
  - sm: 640px
  - md: 768px
  - lg: 1024px
  - xl: 1280px

- No horizontal scrolling allowed
- Typography scales down gracefully on mobile

---

## ⚡ Performance Rules

- No heavy animations libraries unless required
- Optimize images (WebP preferred)
- Avoid large bundle sizes
- Keep UI lightweight and fast

---

## 🚫 Anti-Patterns (DO NOT)

- No overly colorful UI
- No cluttered dashboards on marketing pages
- No inconsistent spacing
- No random font usage
- No overuse of shadows or glow effects

---

## 🎯 Final Rule

If a UI element does not:
- improve clarity
- increase trust
- or increase conversion

👉 it should be removed.
